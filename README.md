## 📖 Описание проекта
Проект представляет собой механику «лутбокса» или мини-слота. Основная задача — реализация бесконечного скролла элементов с плавным разгоном и прецизионной остановкой по центру целевого элемента.
**Ключевые особенности**
- **Бесконечный Reel**: Реализована логика «телепортации» слотов (Wrapping), позволяющая создавать эффект бесконечного вращения с минимальным количеством объектов (4 слота).
- **Precise Stop**: Барабан останавливается не в произвольном месте, а автоматически центрирует ближайший спрайт в маске.
- **Система частиц**: При остановке генерируется эффект, динамически подхватывающий спрайт выпавшей награды.

## 🛠 Технический стек
- **AxGrid Framework * FSM**: Полное управление состояниями (Idle, Spinning, Stopping).
  - **Model**: Реактивное управление данными и состоянием кнопок.
  - **MonoBehaviourExt**: Использование атрибутов `[OnStart]`, `[OnUpdate]` и `[Bind]` для чистоты кода.
- **UI**: Связка через `UIButtonDataBind`, исключающая прямые ссылки в инспекторе.
- **Анимации**: Корутины и математические Easing-функции для плавного торможения.

# 🏗 Архитектура и решения
1. **Состояния FSM**
Логика разделена на независимые классы состояний:
- `IdleState` — ожидание ввода, кнопки активны.
- `SpinningState` — постепенный разгон скорости, блокировка кнопки «Стоп» на первые 3 секунды.
- `StoppingState` — расчет дистанции до «идеального центра» и запуск плавного замедления.

2. **Алгоритм плавной остановки**
Вместо простого уменьшения скорости до нуля, используется расчет целевой дистанции (S):
`S=(StepY⋅Rounds)+OffsetToCenter`
Это гарантирует, что барабан всегда останавливается «пиксель-в-пиксель» по центру, независимо от текущей скорости в момент нажатия кнопки.

4. **Динамические частицы**
Для работы со Sprite Atlas используется модуль Texture Sheet Animation в режиме Sprites. Это позволило избежать создания уникальных материалов для каждого символа и сохранило высокую производительность (один Draw Call для всех частиц).

# 📝 Заметки для ревьюера
В ходе выполнения были решены следующие задачи:
- **Декаплинг**: Логика отображения (`ReelView`) полностью отделена от логики управления (`FSM`), взаимодействие идет через `Settings.Model`.
- **UX/UI**: Реализована блокировка кнопок через `DataBind`, чтобы избежать некорректных переходов состояний.
- **Оптимизация**: Объекты слотов переиспользуются, спрайты объединены в Sprite Atlas.
- **Адаптивность**: Содержимое окна выглядит корректно на всех возможных разрешениях.
